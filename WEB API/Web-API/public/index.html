How it works

Understand the project fast

  1. The server loads local data from seedTuk.json.
  2. Each route returns JSON for one resource type.
  3. The dashboard calls those routes with fetch().
  4. Errors are shown directly in the response viewer.
The UI is intentionally simple: it is here to help you inspect the API structure and the data flow.
